# Implementation Plan: WireGuard Protocol Diagnostic Test

**Branch**: `010-wireguard-test` | **Date**: 2026-07-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/010-wireguard-test/spec.md`

## Summary

Add WireGuard as a fifth protocol diagnostic test (alongside HTTP, HTTPS, STUN, TURN) across Server, CLI, and Android. The server issues short-lived per-session WireGuard credentials (mirroring TURN) and runs an in-process WireGuard peer + UDP echo service — no kernel modules, no root, using wireguard-go with gVisor netstack. The CLI creates two concurrent in-process WireGuard peers from independent credential sets, each sending 512 KB through the server echo service over a 10-second window. Android integrates via a gomobile-generated `.aar` from a new monorepo-local `wgmodule/` using an async JNI callback interface.

**Scale/Scope**: 50 concurrent sessions (configurable); /24 IP pool; 254 usable client IPs

## Key Implementation Notes

### Server WireGuard Service

The `service.go` `Service` struct:
- Holds a single `wireguard-go` `device.Device` backed by `tun.CreateNetTUN()` (gVisor netstack).
- Server has one long-lived Curve25519 private key generated at startup (stored in `Service`, public key returned in all credential responses).
- `SessionManager` manages per-session client keypairs and IP allocation (`next available IP from pool`).
- On session expiry, `device.RemovePeer(clientPublicKey)` is called to clean up the WireGuard peer.
- UDP echo goroutine: `net.ListenPacket("udp", serverTunnelIP:echoPort)` on the `*netstack.Net`, echoes each datagram back to sender.

### CLI WireGuard Test (`cli/diag/wireguard.go`)

```
RunWireGuard(cfg, stack, timeout, spinner):
  1. FetchWireGuardCredentials(A) + FetchWireGuardCredentials(B)  [parallel]
  2. CreatePeer(credsA) + CreatePeer(credsB)                      [parallel]
  3. WaitHandshake (≤5s) for both peers                           [parallel, timeout]
  4. Transfer window (10s):
       goroutineA: send 512 KB UDP datagrams to echo port, receive echoes, measure RTT
       goroutineB: same concurrently
  5. Close both peers
  6. Aggregate metrics → TestResult
```

### Android Integration

- `WireGuardTest.kt` calls `Wgmodule.runWireGuardTestAsync(credsJSON, callback)` (gomobile-generated package name `wgmodule`).
- `callback.onResult(result, errMsg)` is delivered on a goroutine thread — `withContext(Dispatchers.Main)` wraps the callback to route to the Kotlin coroutine context.
- `DiagnosticRunner` fetches credentials via HTTP (same pattern as STUN/TURN config fetch) then calls `runWireGuardTest()`.

The `wglib.aar` is committed to the repo (pre-built) so Android developers do not need the Go toolchain installed. CI regenerates it when `wgmodule/` changes.
